\subsection{Detailed Description}
Main namespace. 

This is where all the iteam-related functions are and will be declared, this is so it doesn't interfere with GP2D functions, variables or similar, unless explicitly stated.

This sets the screen resolution (for \doxyref{iteam}{p.}{namespaceiteam} use), some variables and starts the GP2D system. Initializes video, audio, sets the Max. FPS to 60 and sets SDL to run SDL\_\-Quit when our game ends or gets closed. \index{iteam@{iteam}!Load@{Load}}

\subsubsection{\setlength{\rightskip}{0pt plus 5cm}int iteam::Load ()}\label{namespaceiteam_e3ea8db4c870a297a9cd5bca9dfb7a4f}


Loads fonts, graphics, sounds, players, backgrounds, etc and sets them up on the scene (this should change). 

This function creates the camera, loads game fonts, music, sound, adds two players, loads the terrain and its background layers, it loads the GUI and a countdown sprite. It also initializes the weapon subsystem by Mike. This should change too, so \doxyref{Load()}{p.}{namespaceiteam_e3ea8db4c870a297a9cd5bca9dfb7a4f} would be used when a new game starts, and load everything up in a different way. Or maybe a preloading function when the game starts for the GUI \& fonts and other stuff wouldn't be bad, and just to load the players and terrain when a new fight comes in.
